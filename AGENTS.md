## Architectural Guidelines

### Core Principles
1.  **Framework:** Use `optimistix` for the optimization loop structure (`AbstractMinimiser`). The solver implements the `optimistix.AbstractMinimiser` interface and runs through `optimistix.minimise`.
2.  **Differentiation:** Use `jax.grad`, `jax.jacrev`, and `jax.jvp` for derivatives. Do not approximate derivatives numerically unless strictly necessary.
3.  **Typing:** Strictly use `jaxtyping` with `beartype` decorators for all functions.
4.  **Functional:** All logic must be pure functions. No side effects. Use `jax.lax.scan` or `jax.lax.while_loop` for loops.

### Scaling Considerations: L-BFGS over BFGS

Classical SLSQP maintains a **dense** n×n BFGS approximation to the Hessian of the Lagrangian. This requires O(n²) memory and O(n²) work per iteration. For target problem sizes (5,000–50,000 variables), this is prohibitive:

| n | Dense Hessian memory | L-BFGS memory (k=10) |
|---|---|---|
| 1,000 | 8 MB | 160 KB |
| 10,000 | 800 MB | 1.6 MB |
| 50,000 | 20 GB | 8 MB |

L-BFGS stores only the last k step/gradient-difference pairs (s_i, y_i) and computes Hessian-vector products in O(kn) time using the compact representation:

```
B_k = γI - W N⁻¹ Wᵀ
```

where W = (γS, Y) and N is a small 2k×2k matrix. VARCHEN-style B0 damping (Lotfi et al., 2020) is applied to each curvature pair before storage, damping toward `B0 = diag(diagonal)` instead of the full L-BFGS approximation B.  This is O(n) instead of O(k²n), always well-conditioned, and avoids the circular dependency where a badly-conditioned B poisons its own damping.  The damping threshold is configurable via `damping_threshold` on `SLSQP` (default 0.2); setting it to 0.0 disables damping entirely.  The solver tracks the condition number `kappa(B0) = max(d)/min(d)` and performs a soft reset (keep most recent pair, drop rest) when kappa exceeds 1e6.

### Scaling Considerations: Projected CG over Dense KKT

Classical SLSQP solves the QP subproblem by forming and factorising the (n+m)×(n+m) dense KKT system at O(n³) cost. This implementation uses **projected conjugate gradient** (CG) inside an active-set loop:

1. **Projection**: For active constraints with matrix A (m_active × n), define the null-space projector P(v) = v - Aᵀ(AAᵀ)⁻¹Av. The AAᵀ system is only m_active × m_active (tiny, since m << n).
2. **CG in null space**: Run conjugate gradient on the projected system. Each iteration requires one HVP (O(kn) for L-BFGS) and one projection (O(mn)).
3. **Active-set outer loop**: Add most violated inequality or drop most negative multiplier until KKT conditions are satisfied.

Total cost per QP solve: O(n·k·t), where t is the number of CG iterations (typically t << n).

### Pluggable Inner QP Solvers

The equality-constrained subproblem at each active-set iteration is delegated to a pluggable **inner solver**. All strategies implement the `AbstractInnerSolver` interface (`eqx.Module`) defined in `inner_solver.py` and return an `InnerSolveResult(d, multipliers, converged)`. The active-set logic in `qp_solver.py` is agnostic to the inner solver strategy: it receives the `inner_solver` instance and calls `inner_solver.solve(...)`.

Three strategies are available:

- **`ProjectedCGCholesky`** (default): Cholesky-based null-space projected CG. Factorises `AAᵀ + εI` once via `jnp.linalg.cholesky`, reuses for all CG iterations and multiplier recovery with iterative refinement. Cost: O(m³) factorization + O(kn + m²) per CG step.

- **`ProjectedCGCraig`**: Projected CG with CRAIG's method (Golub-Kahan bidiagonalization) replacing Cholesky. Each projection iteratively computes `x = Aᵀ(AAᵀ)⁻¹b` without forming `AAᵀ`, eliminating the O(m³) factorization and the ε regularisation. Better equality constraint satisfaction (no regularisation bias). Multiplier recovery via CG on normal equations `AAᵀ λ = A(Bd+g)` uses the separate `mult_recovery_tol` (default 1e-12) and `mult_recovery_max_iter` (default 200) so multipliers can be solved tighter than the CRAIG projection without blowing up outer-solve cost. The CRAIG iteration flags **breakdown** when either `alpha` or `beta` falls below `_CRAIG_BREAKDOWN_TOL = 1e-14`, which is surfaced through the returned `converged=False` — this is the rank-deficient / inconsistent-rhs signal and must not be silently clamped away. Each step also applies one round of partial re-orthogonalisation against the latest `v_k` and `u_{k+1}` to keep the short recurrence honest under floating-point drift. Cost: O(mn · t_craig) per projection. Reference: Craig (1981); Golub & Kahan (1965).

- **`MinresQLPSolver`**: Solves the full (n+m)×(n+m) saddle-point KKT system `[B Aᵀ; A 0][d; λ] = [-g; b]` directly using Preconditioned MINRES-QLP (PMINRES-QLP, Table 3.5 of Choi 2006). No null-space projection or separate multiplier recovery needed — d and λ come out simultaneously. Handles indefinite/singular systems natively. All iterations use QLP mode (both left QR and right QLP orthogonalizations) with numerically stable SymOrtho rotations (Table 2.9). The solver also carries an explicit **Lanczos-breakdown short-circuit** (`betan_new < 1e-30 * max(beta1, 1)`): when the recurrence collapses we freeze the iterate and return `converged=False` so the QP layer triggers its fallback path instead of amplifying noise through the remaining iterations. Uses a **block-diagonal SPD preconditioner** `M⁻¹ = diag(B_diag⁻¹, S⁻¹)` where `B_diag⁻¹` is the L-BFGS inverse Hessian diagonal and `S = A B_diag⁻¹ Aᵀ` is the Schur complement. This satisfies the SPD preconditioner requirement from Section 3.4 of Choi (2006). When the bound-fixing loop supplies a `free_mask`, the preconditioner's primal block is masked to the free subspace (`v → free * B⁻¹(free * v)`) to prevent L-BFGS cross-coupling from leaking into the zero dimensions of the reduced KKT system; without this masking the preconditioned Lanczos vectors become contaminated and constraint satisfaction degrades. The `_solve_kkt_minres_qlp` epilogue additionally re-applies `d := free * d + d_fixed` on both the preconditioned and unpreconditioned paths so that floating-point drift cannot leak mass into variables that were explicitly fixed at a bound. **Posterior M-metric range-space projection:** PMINRES-QLP minimises only the *Euclidean* residual of the full KKT system; the constraint subvector `||A d - b||` carries no separate guarantee. Truncation by `max_iter` or stalling under the SPD preconditioner therefore leaves the SLSQP step infeasible, and on bound-heavy problems (e.g. `Portfolio(n=5000)`) the active-set churn that follows is the "MINRES diverges" failure mode. After the PMINRES-QLP iteration completes, an **M-metric projection** `δd = M Aᵀ (A M Aᵀ)⁻¹ (A d - b);  d ← d − δd` is applied. This is the closest correction to `d` in the metric `M⁻¹` and the unique minimiser of `||δd||_{M⁻¹}` subject to `A(d − δd) = b`. The factorisation of `A M Aᵀ + 1e-8·I` is **already computed for the Schur block of the preconditioner**, so the projection costs only one extra back-solve plus a matvec — essentially one additional Lanczos step. A single shot empirically reduces `||A d - b||` by 4–7 orders of magnitude (residual floor `O(eps · cond(A M Aᵀ) · ||A d_raw - b||)` from the `1e-8·I` Schur regularisation); enough that on `Portfolio(n=5000) + use_exact_hvp_in_qp=True` the solver converges in 3.1 s where every other configuration stalls. To eliminate even that floor on ill-conditioned problems, the projection is wrapped in a fixed-point **iterative refinement** loop (HR Algorithm 4.18 step 1(a) applied to the projection rather than the Krylov solve): up to `proj_refine_max_iter` (default `3`) rounds of `d ← d − M Aᵀ (A M Aᵀ)⁻¹ (A d - b)` with early termination once `||A d - b|| <= proj_refine_atol + proj_refine_rtol · (1 + ||b||)` (defaults `1e-14` and `1e-10`). Each round is one matvec plus one cached Schur back-solve — no refactorisation. The actual rounds used and the final residual are surfaced through `InnerSolveResult.proj_residual` / `n_proj_refinements`, propagated to `QPResult`, and accumulated into `SLSQPDiagnostics.max_proj_residual` / `n_proj_refinements`. When no SPD preconditioner is supplied, a small dedicated `A Aᵀ + ε·I` Cholesky is built just for the 2-norm projection (same refinement loop). Multipliers are left as PMINRES-QLP returned them; the residual stationarity inconsistency `O(||B|| · ||δd||)` is absorbed by the outer L1 merit and L-BFGS secant blending. Cost: O((n+m) · t_minres) per solve plus `(1 + n_refine)` triangular `m×m` solves. Reference: Choi (2006), PhD thesis, Stanford; Choi, Paige & Saunders, SIAM J. Sci. Comput. 33(4), 2011; Orban & Arioli, *Projected Krylov Methods for Saddle-Point Systems*, SIAM J. Matrix Anal. Appl. 36(3), 2014, Lemma 3.1; Benzi, Golub & Liesen, *Numerical Solution of Saddle Point Problems*, Acta Numerica, 2005, §5; Heinkenschloss & Ridzal, *A Matrix-Free Trust-Region SQP Method for Equality Constrained Optimization*, SIAM J. Optim. 24(3), 2014, Algorithm 4.18.

The default `ProjectedCGCholesky` is constructed automatically if no `inner_solver` is passed to `solve_qp`. Users can pass a different strategy via the `inner_solver` parameter on `SLSQP` (forwarded to `solve_qp` and the bound-fixing loop in `solver.py`).

**Adaptive tolerance threading.** When `adaptive_cg_tol=True`, the Eisenstat-Walker tolerance computed by the outer solver is threaded through `solve_qp` and the bound-fixing loop to the inner solver via the `adaptive_tol` parameter on `AbstractInnerSolver.solve()`. `ProjectedCGCholesky` and `ProjectedCGCraig` use it to override their instance tolerance; `MinresQLPSolver` ignores it because constraint satisfaction in the full KKT system degrades when the MINRES residual threshold is loosened (unlike null-space CG where constraints are enforced by the projector regardless of CG tolerance).

**Preconditioning**: The CG solver supports an optional preconditioner M ≈ B̃⁻¹. Two preconditioner types are available, controlled by `preconditioner_type` on `SLSQP`:

- **`"lbfgs"` (default)**: L-BFGS inverse Hessian (two-loop recursion, Algorithm 7.4 of Nocedal & Wright). When equality constraints are present *and* sSQP is active (`proximal_tau > 0`), the preconditioner is upgraded to B̃⁻¹ via the Woodbury identity so it matches the actual QP system matrix B̃ = B + (1/μ) A_eq^T A_eq, where μ is the adaptive proximal parameter. When sSQP is disabled (`proximal_tau = 0`), the plain B⁻¹ preconditioner is used.

- **`"diagonal"`**: Stochastic Hessian diagonal estimate (Bekas, Kokiopoulou & Saad, 2007). Probes the exact Lagrangian HVP with `diagonal_n_probes` (default 20) Rademacher random vectors to estimate diag(H_L). Each probe costs one HVP evaluation. The preconditioner is M⁻¹ = diag(1/d̂) with small/negative entries clamped to a positive floor. Requires an exact HVP (`use_exact_hvp_in_qp=True` or `obj_hvp_fn` provided). When the proximal term is active, the Woodbury correction is applied with D = diag(d̂) instead of B. This preconditioner is independent of L-BFGS history quality, making it robust on ill-conditioned problems where L-BFGS resets degrade the preconditioner.

For projected CG, the **constraint preconditioner** (Gould, Hribar & Nocedal, SIAM J. Sci. Comput., 2001) is used: `z = M r - M A^T (A M A^T)^{-1} A M r`. This solves the augmented saddle-point system and preserves the M^{-1}-inner product in null(A), unlike the naive P(M(r)) which destroys preconditioning quality for ill-conditioned problems. The cost is m extra preconditioner applications to form M A^T, amortized over all CG iterations.

**Newton-CG mode** (`use_exact_hvp_in_qp=True`): Replaces the frozen L-BFGS HVP with the exact Lagrangian HVP (via AD) in the CG inner loop. Each CG step costs one forward-over-reverse pass. L-BFGS is still updated for preconditioning and as a fallback. When the user provides `obj_hvp_fn`, that is used; otherwise the solver auto-computes the objective HVP via `jax.jvp(jax.grad(f), ...)`. This is the standard Newton-CG approach (KNITRO, Ipopt) and dramatically improves convergence on ill-conditioned problems where L-BFGS cannot capture extreme eigenvalue spread.

**CG regularization**: The CG curvature guard uses a scale-invariant relative threshold `pBp <= delta^2 * ||p||^2` instead of a hard absolute one. This prevents false negative-curvature detection when `||p||` is small (e.g. after an L-BFGS diagonal reset with high condition number). The step length and residual use the true curvature, so the CG solution is unbiased. Controlled by `cg_regularization` on `SLSQP` (default 1e-6). Based on SNOPT Section 4.5 (Gill, Murray & Saunders, 2005).

**Cholesky-based projection** (applies to `ProjectedCGCholesky` only): The AAᵀ factorization used in the null-space projector is computed once via `jnp.linalg.cholesky` and reused for all CG iterations via `cho_solve`. This amortizes the O(m³) factorization cost: each subsequent projection solve is O(m²) instead of O(m³), giving a significant speedup when the CG loop runs many iterations. The Cholesky is regularized with `1e-8 * I` for numerical stability; after multiplier recovery, one step of **iterative refinement** corrects the O(eps · cond(AAᵀ)) error introduced by the regularization, squaring the relative error (e.g. from ~1e-5 to ~1e-10). This prevents stagnation at near-KKT points where the Lagrangian gradient residual from imprecise multipliers exceeds the outer convergence threshold.

**Bound separation via variable fixing**: Box constraints (x_lower ≤ x ≤ x_upper) are handled separately from general inequality constraints to avoid inflating the projection matrix. The QP solver operates on general equality and inequality constraints only (m_eq + m_gen rows), keeping the projection cost at O((m_eq + m_gen)³). After the QP solve, an iterative bound-fixing loop identifies variables whose QP direction violates bounds, fixes them at their bound values, and re-solves the CG in the reduced free-variable subspace. The loop also checks bound multiplier signs and releases variables with wrong-sign multipliers (indicating the variable wants to move away from its bound). This active-set iteration for bounds typically converges in 2-5 passes and is capped at 5 iterations. The inner solver's `solve()` method supports `free_mask` and `d_fixed` parameters: fixed variables are held constant while the solver optimizes over the free subspace, with the effective gradient shifted to account for cross-coupling between fixed and free variables.

After the line search, the bound block of `state.multipliers_ineq` is **recomputed at `x_{k+1}` from the partial Lagrangian gradient** instead of being trusted as the QP-level snapshot at `x_k`. The QP-level recovery `B d + g − A^T λ` inherits three error sources: the L-BFGS HVP `B ≠ H`, the recovery is at `x_k` not `x_{k+1}`, and the active-bound mask is read from the QP direction rather than from `x_{k+1}` (so an `α < 1` line search leaves the recovered multiplier "for the wrong point"). On bound-heavy problems (e.g. `Portfolio(n=5000)` with `MinresQLPSolver`, where the optimum has thousands of active bounds) this used to pin `||∇L|| / |L|` above `rtol` even when the constraints were satisfied and `α = 1`. The post-line-search recovery in `_recover_bound_multipliers` (called from `step()`) reads off the partial Lagrangian gradient `∇f(x_{k+1}) − J_eq^T λ − J_gen^T μ_gen` (built via `compute_partial_lagrangian_gradient` from `slsqp_jax/hessian.py`) at the indices that satisfy the at-bound test `(y_new - lb) <= 1e-12 + eps · (1 + |y_new|)` (and analogously for upper bounds), clamps to `≥ 0` for dual feasibility, and splices the result directly into `blended_mult_ineq[m_general:]`. The same vector then feeds both the `grad_lagrangian_new` computation (the stationarity check) and the `grad_lagrangian_old` computation that builds the L-BFGS secant pair `y_k = ∇L_new − ∇L_old`. Because the bound Jacobian `state.bound_jac` is constant (identity-style rows for lower bounds, their negatives for upper bounds), the bound contribution to `y_k` cancels identically — `[J_bound − J_bound]^T μ_bound ≡ 0` regardless of `μ_bound` — so the splice leaves the secant condition algebraically unchanged. This matches the L-BFGS-B precedent (Byrd, Lu, Nocedal & Zhu, SIAM J. Sci. Comput. 16(5), 1995; Zhu, Byrd, Lu & Nocedal, ACM TOMS 23(4), 1997, Algorithm 778) where `y_k = ∇f(x_{k+1}) − ∇f(x_k)` (i.e. `μ_bound = 0` at both endpoints) is used for bound-constrained problems. The corrected bound multipliers are also written back into `new_state.multipliers_ineq` (the general portion is taken from `qp_result.multipliers_ineq` for symmetry with `multipliers_eq`), so LPEC-A's predictor (`compute_rho_bar` consumes `state.multipliers_ineq`), the verbose diagnostics, and the next QP's warm-start all see the sharper values. There is no user-facing parameter: the recovery is always on whenever bounds are present and silently degrades to a no-op when `bounds is None` (or when no finite lower/upper bounds exist).o account for cross-coupling between fixed and free variables. When LPEC-A is enabled (`active_set_method` in `"lpeca_init" / "lpeca"`) and `lpeca_predict_bounds=True` (default), the bound portion of LPEC-A's predicted active set (which spans the full inequality vector `[general; lower_bound; upper_bound]`) is scattered onto variable indices and installed as the *initial* `free_mask` / `d_fixed` for the bound-fixing loop, warm-starting the active bound set on bound-dominated problems. The first loop pass is forced to run the reduced-space inner solve so the warm-started mask is actually consumed; subsequent passes refine it as usual. See the **Bound-constraint extension** bullet under the LPEC-A section below for the detailed gating logic.

### Derivative Computation

By default, the solver computes:
- **Objective gradient** via `jax.grad` (reverse-mode)
- **Constraint Jacobians** via `jax.jacrev` (reverse-mode, O(m) passes — faster than jacfwd's O(n) passes when m << n)
- **HVP fallback** via `jax.jvp(jax.grad(f), ...)` (forward-over-reverse)

Users can supply custom derivative functions to:
- Bypass automatic differentiation when functions contain `while_loop` or other non-reverse-differentiable operations
- Avoid redundant computation
- Handle functions that are not reverse-mode differentiable

### Reverse-mode AD Limitations

JAX's reverse-mode AD does not support differentiating through `jax.lax.while_loop` or variable-length control flow. If user functions contain such operations, they must supply custom derivative functions via the optional fields on `SLSQP`:
- `obj_grad_fn` - bypasses `jax.grad`
- `eq_jac_fn`, `ineq_jac_fn` - bypasses `jax.jacrev`
- `obj_hvp_fn`, `eq_hvp_fn`, `ineq_hvp_fn` - bypasses forward-over-reverse

### Algorithm Structure

Each SLSQP iteration performs four steps:
1. **QP subproblem**: Construct quadratic approximation of objective and linearise constraints. Solve QP to obtain search direction.
2. **Line search**: Use Han-Powell L1 merit function with backtracking Armijo conditions.
3. **Accept step**: Update iterate x_{k+1} = x_k + α·d_k.
4. **Hessian update**: Append new curvature pair (s, y) to L-BFGS history (or skip if using exact HVPs). The gradient difference `y_k = ∇_x L(x_{k+1}, λ_{k+1}) − ∇_x L(x_k, λ_{k+1})` uses the **same** (blended) multipliers at both points to satisfy the secant condition (Nocedal & Wright §18.3). The initial Hessian uses **component-wise secant diagonal scaling**: `B₀ = diag(d)` where `d_i = |y_i · s_i| / (s_i² + ε)`, clipped to `[1e-2, 1e6]`. This gives `d_i ≈ |H_{ii}|` for diagonal Hessians regardless of the step direction. The classical Shanno-Phua formula `d_i = y_i² / (yᵀs)` uses a single scalar normalizer, producing `d ∝ h_i²` for multi-component steps, which severely underestimates curvature and causes 10-100x slowdowns on problems with moderate condition numbers.

### Robustness: Anti-cycling and Stagnation Detection

**QP anti-cycling (EXPAND procedure):** The active-set QP solver uses the EXPAND procedure (Gill, Murray, Saunders & Wright, 1989) to prevent cycling at degenerate vertices. The working tolerance ramps monotonically as `delta_k = tol + k * tau` with `tau = tol * expand_factor / max_iter`. With the default `expand_factor=1.0`, `delta_k` grows from `tol` at `k=0` to `2 * tol` at `k=max_iter`, which dislodges floating-point ties at degenerate vertices while keeping the QP's working-set decisions coherent with the outer SLSQP stationarity tolerance. Controlled by the `expand_factor` parameter on `solve_qp`.

**Noise-aware multiplier drop:** Multipliers recovered from the projected-CG inner solver carry numerical noise (typically `O(eps * cond(AAᵀ))`). To prevent a noise-flipped multiplier from spuriously dropping a genuinely active constraint (and triggering an immediate re-activation on the next iteration), the negative-multiplier drop test uses `lambda_i < -max(working_tol, mult_drop_floor)`. The `mult_drop_floor` parameter on `solve_qp` (default `1e-6`) sets the noise floor.

**Ping-pong detector (opt-in):** Even with EXPAND and the noise floor, pathological degeneracies can still cause add-then-drop oscillations on the same constraint. The active-set loop tracks `last_add_idx` and `last_drop_idx` and increments a `ping_pong_count` whenever the same index is added immediately after being dropped (or vice versa). When the count reaches `qp_ping_pong_threshold` the loop short-circuits with `converged=True` and `ping_ponged=True`. The detector is **disabled by default** (`qp_ping_pong_threshold = 2**31 - 1`): production paths rely on the classical behaviour where an oscillating pair keeps iterating until `qp_max_iter` is reached, and short-circuiting with the last inner-solver multipliers has been observed to leak ill-conditioned coupled multipliers into the outer merit penalty (especially under `MinresQLPSolver`). Users with confirmed degenerate vertices and a bounded merit stream can opt in by setting the threshold to `3`–`8`. The diagnostics counter `n_qp_ping_pong` and the per-step `qp_cyc` column in the verbose output still record how often the mechanism would fire.

**LPEC-A active set identification:** As an alternative to (or complement of) EXPAND, the solver supports LPEC-A (Oberlin & Wright, 2005, Section 3.3) for predicting the active constraint set before the QP active-set loop. LPEC-A computes a proximity measure `rho_bar` from primal constraint values, dual multipliers, and the stationarity residual, then applies a threshold test `c_ineq_i <= (beta * rho_bar)^sigma` to predict active constraints. Under MFCQ + second-order sufficiency, the prediction is asymptotically exact (including weakly active constraints). Controlled by `active_set_method` on `SLSQP`: `"expand"` (default, no LPEC-A), `"lpeca_init"` (LPEC-A warm-start + EXPAND fallback), or `"lpeca"` (LPEC-A + fixed tolerance, no EXPAND growth). Threshold parameters: `lpeca_sigma` (default 0.9), `lpeca_beta` (default `1/(m_ineq + n + m_eq)`).

LPEC-A's asymptotic correctness only holds when `rho_bar` is small enough for the iterate to be near a KKT point. The implementation guards against the over-saturation that occurs far from the solution with three layers:

1. **SQP warm-up gate (`lpeca_warmup_steps`, default 3):** LPEC-A is bypassed entirely for the first few outer iterations, when the multiplier estimates fed into `rho_bar` are still transient. Counted by `n_lpeca_bypassed`.
2. **Trust gate (`lpeca_trust_threshold`, default 1.0):** when `rho_bar > lpeca_trust_threshold` the prediction is replaced with an empty set. This replaces the previous `min(threshold, max|c_ineq|)` clamp, which was mathematically vacuous (every `c_i` is bounded by `max|c_j|` by construction) and trivially flagged every inequality as active early in SQP. Also counted by `n_lpeca_bypassed`.
3. **Rank-aware size cap:** even when the trust gate passes, the prediction is truncated so at most `n - m_eq - 1` inequalities are predicted active, preserving a LICQ-like rank margin in the working-set Jacobian. The most-violated constraints (smallest `c_ineq_i`) are kept. Counted by `n_lpeca_capped`; flagged per step by `lpeca_capped` on `QPResult`.

**Bound-constraint extension (`lpeca_predict_bounds`, default `True`):** LPEC-A's prediction already covers the full inequality vector, which includes the lower/upper bound rows appended by the bound-handling machinery. When `lpeca_predict_bounds=True`, the bound portion of the predicted active set is split into lower/upper components, scattered back onto the variable indices using the precomputed `_lower_indices` / `_upper_indices` tuples, and installed as the initial `free_mask` and `d_fixed` for the bound-fixing loop in `_solve_qp_subproblem`. The first pass of that loop is forced to run the reduced-space inner solve so the QP direction and multipliers become consistent with the warm-started active bounds; subsequent passes use the standard add/drop logic. When both the lower- and upper-bound rows predict the same variable (rare, can only happen near `lb ≈ ub`), the lower bound takes precedence. The warm-up and trust gates apply before the bound prediction is extracted, so a bypassed LPEC-A cleanly reverts to the cold-started bound-fixing loop. The per-step variable count is exposed on `QPResult.n_lpeca_bounds_prefixed` and accumulated in the `SLSQPDiagnostics.n_lpeca_bounds_prefixed` counter. Setting `lpeca_predict_bounds=False` restricts LPEC-A to general inequalities only and reverts to the previous bound-fixing behavior.

Optional LP refinement via `mpax.r2HPDHG` when `lpeca_use_lp=True` (requires `mpax` from the `extras` dependency group).

**QP failure diagnostics:** The `SLSQPDiagnostics` block accumulates several QP-failure counters across the SQP run: `n_qp_failures` (QP did not converge), `n_qp_budget_exhausted` (active-set loop reached `max_iter`), `n_qp_ping_pong`, `n_lpeca_bypassed`, `n_lpeca_capped`, `n_lpeca_bounds_prefixed`, plus the high-water marks `max_qp_iterations` and `max_qp_active_size`. The verbose output appends two columns per SQP step — `QPiter` (final active-set iteration count) and `QPcyc` (bitmask: `0` clean, `1` ping-pong, `2` budget exhausted, `3` both) — to surface QP behavior without printing the full QP state.

**Multiplier stability:** Wright (SIAM J. Optim., 2002, Theorem 5.3) proved that SQP superlinear convergence requires multiplier stability across iterations. The solver promotes this through: (1) warm-starting the QP active set from the previous iteration, (2) an adaptive EXPAND tolerance tied to the outer KKT residual, (3) adaptive proximal multiplier stabilization (sSQP, active for equality-constrained problems when `proximal_tau > 0`, with `mu = clip(kkt_residual^tau, mu_min, mu_max)` per Wright eq 6.6; controlled by `proximal_tau`, `proximal_mu_min`, and `proximal_mu_max` on `SLSQP`), and (4) alpha-scaled multiplier blending (`lambda_new = lambda_prev + alpha * (lambda_QP - lambda_prev)`) for L-BFGS secant pair consistency when the line search accepts a partial step. Raw QP multipliers are stored in the state for convergence checking. The ceiling `mu_max` (default 0.1) prevents weak equality enforcement when the KKT residual is large (Wright's local theory assumes eta < 1). Setting `proximal_tau = 0` disables sSQP entirely: equality constraints are enforced via direct null-space projection in the QP subproblem instead of the augmented Lagrangian penalty. This avoids the ill-conditioning introduced by the `(1/μ) A_eq^T A_eq` term, which can be beneficial for well-conditioned problems or when cross-platform reproducibility is important.

**L-BFGS reset strategies (VARCHEN-inspired):** The L-BFGS initial Hessian is stored as a per-variable diagonal `B_0 = diag(d)` using component-wise secant scaling (`d_i = |y_i · s_i| / (s_i² + ε)`). The diagonal is clipped to `[lbfgs_diag_floor, lbfgs_diag_ceil]` (default `[1e-4, 1e6]`) in both `lbfgs_append` and `lbfgs_reset`, so cold starts, soft resets, and ordinary updates share the same clipping regime. The floor is deliberately larger than machine epsilon: coordinates with `|y_i · s_i| / s_i² < 1e-4` either indicate a near-zero secant step or vanishing local curvature, both of which are better handled by re-estimating from the next pair than by extrapolating an inverse close to a division by zero. The skip heuristic `(relative_curvature < 1e-8) & (sTy_raw > 0)` only skips pairs with positive but extremely tiny curvature; pairs with `sTy_raw <= 0` are handled by the separate VARCHEN damping. The reset chain follows an escalating strategy: (1) **soft reset** (keep most recent pair, drop rest) triggered by `kappa(B0) > 1e6` after each append, or on QP/line-search failure; (2) **identity reset** (`B_0 = I`) after `qp_failure_patience` (default 3) consecutive QP failures or `ls_failure_patience` (default 3) consecutive line search failures. The soft reset (VARCHEN Algorithm 1, Step 7) preserves the most relevant curvature pair, avoiding the catastrophic information loss from the old SNOPT diagonal/identity reset chain.

**QP failure recovery:** When the QP fails to converge, the solver (1) gates penalty parameter updates so unreliable multipliers cannot permanently inflate `rho`, and (2) falls back to a **projected steepest descent direction** `P(-grad_f)` where P projects onto `null(J_eq)`. This ensures the fallback direction satisfies `J_eq . d = 0`, preventing catastrophic constraint violation (the unprojected `-grad_f` can have `sum(d) = O(n)` for a simplex constraint, making the L1 merit DD massively positive and the line search unable to find a step). The QP solver also guards against false convergence: when the active-set loop reaches `max_iter`, the convergence flag is forced to `False` regardless of the EXPAND-relaxed tolerance state. Combined with the L-BFGS soft reset, this prevents the cascade where one QP failure leads to permanent stagnation.

**L1 merit directional derivative:** The backtracking line search uses the proper L1 merit directional derivative `D_phi = grad_f . d + rho * sum_i sign(c_eq_i) * (J_eq d)_i - rho * sum_{j: c_ineq_j < 0} (J_ineq d)_j` instead of the simpler `grad_f . d`. When the QP direction satisfies `J_eq d = -c_eq`, this simplifies to `grad_f . d - rho * ||c_eq||_1`, making the Armijo condition easier to satisfy. Constraint Jacobians are passed from the solver to the line search.

**Line search failure recovery:** When the QP converges but the resulting direction is not a descent direction for the L1 merit function (typically due to a poorly conditioned L-BFGS Hessian), the backtracking line search exhausts its iteration budget and returns a tiny step size. The solver tracks consecutive line search failures and applies the escalating L-BFGS reset strategy: soft reset on each failure, identity reset after `ls_failure_patience` (default 3) consecutive failures. Controlled by `ls_failure_patience` on `SLSQP`. If `consecutive_ls_failures` reaches `2 * ls_failure_patience`, the solver flags the run as fatally stagnant (`state.ls_fatal = True`) and `terminate()` routes it to `RESULTS.nonlinear_divergence`. The old `qp_optimal & primal_feasible` success disjunct has been removed because `qp_optimal` latches on any sequence of zero-norm steps — exactly what a collapsed line search produces — so it was masking chronic LS failures as successful convergence. Convergence now strictly requires both relative stationarity and primal feasibility; `qp_optimal` is retained only as a diagnostic flag.

**Zero-step convergence detection:** When the QP solver repeatedly converges with `||d|| < atol`, the current point satisfies the QP's KKT conditions (within `cg_tol`) even though the outer stationarity criterion `||∇L|| <= rtol * max(|L|, 1)` may not be met — typically because of residual multiplier imprecision from the regularized Cholesky projection. After `zero_step_patience` (default 3) consecutive zero-step iterations, the solver declares `successful` convergence rather than waiting for the merit-based stagnation counter (which would report `nonlinear_divergence`). Controlled by `zero_step_patience` on `SLSQP`. The zero-step check has two stages: a pre-line-search check (`||d|| < atol`) for null-space CG solvers that return d=0 exactly, and a post-line-search check (`α·||d|| < atol`) for full-KKT solvers like `MinresQLPSolver` that always return `||d|| > 0` (the coupled (d,λ) system has a non-trivial solution even when the projected gradient is negligible). The post-line-search check detects that the line search could only accept a tiny α, making the effective step negligible.

**Outer-loop stagnation detection:** The solver uses a merit-based patience counter with patience `W = max_steps // 10`. At each step, it computes the L1 merit `φ = f + ρ·(‖c_eq‖₁ + ‖max(0,−c_ineq)‖₁)` and checks whether `φ_new < best_φ − stagnation_tol · max(|best_φ|, 1)`. If yes, `best_φ` is updated and the counter resets. Otherwise, `steps_without_improvement` increments. Stagnation fires when `steps_without_improvement >= W` and `step_count >= W`, terminating early with `nonlinear_divergence`. This is O(1) state (vs O(Wn) for the old x-history ring buffer) and directly measures optimization progress. Controlled by `stagnation_tol` on `SLSQP`.

**Best-iterate divergence rollback:** As a final guardrail against runaway QPs (e.g. truncated PMINRES-QLP that returns a non-feasible direction the line search cannot rescue), `SLSQPState` carries `best_x`, `blowup_count` and `diverging`. At each `step()`, after the merit `φ_new` is computed, `best_x` is updated to the current iterate when `merit_new < best_merit − stagnation_tol · max(|best_merit|, 1)` (same predicate used by the stagnation counter, so the two stay in sync). A *blowup* event is detected when `(merit_new − best_merit) > divergence_factor · max(|best_merit|, 1)` *or* `merit_new` is non-finite; consecutive blowups increment `blowup_count`, otherwise it resets to 0. Once `blowup_count >= divergence_patience`, `diverging` latches `True`, the iterate returned by `step()` is overwritten with `state.best_x` (cheap rollback — no full state checkpoint), and `terminate()` includes `state.diverging` in the disjunction so the run finishes with `RESULTS.nonlinear_divergence`. The L-BFGS history, multipliers and gradients still reflect the *attempted* step; only the iterate the driver sees is rolled back, which is enough because the loop terminates immediately after. Controlled by `divergence_factor` (default `10.0`) and `divergence_patience` (default `3`) on `SLSQP`. The diagnostics block carries `n_divergence_blowups` (cumulative count of blowup-counter increments, whether or not the trigger fired) and `divergence_triggered` (latched `True` once rollback fires). Inspired by Heinkenschloss & Ridzal (2014, §4.3) — the spirit of their normal-step over-correction control applied to a line-search SQP.

**Convergence criterion:** The solver declares convergence when both conditions hold (and at least `min_steps` iterations have elapsed): (1) **relative stationarity** `||∇_x L|| <= rtol * max(|L|, 1)` where `L = f − λ_eq^T c_eq − μ_ineq^T c_ineq` is the Lagrangian value, and (2) **primal feasibility** `max|c_eq| <= atol` and `max(0, −c_ineq) <= atol`. The relative stationarity criterion is motivated by floating-point arithmetic: when the gradient is negligible relative to `|L|`, a step of that magnitude cannot change `L` in finite precision. The `max(|L|, 1)` safeguard prevents the criterion from becoming vacuous when `L ≈ 0`. `rtol` controls stationarity (default 1e-6); `atol` controls feasibility (default 1e-6).

**QP inner-solve failure propagation:** `QPResult.converged` is set to `False` only when the resulting direction is non-finite (NaN/Inf). Inner solvers that merely exhaust their CG/MINRES iteration budget — which is common on the proximal stabilized system `B + (1/mu) A_eq^T A_eq` for tight `cg_tol` — still return a usable (finite) descent direction, and for an outer SQP an imprecise Newton direction is almost always better than the projected-gradient fallback. Flagging truncated CG as a hard failure caused the solver to soft-reset the L-BFGS history and fall back to steepest descent on every outer step, breaking convergence on otherwise benign problems. Inner non-convergence is instead tracked via `SLSQPDiagnostics.n_qp_inner_failures` for post-hoc analysis.

**Diagnostics:** `SLSQPState` carries an `SLSQPDiagnostics` (`eqx.Module`) accumulator populated branch-free inside `step()`. Fields include `n_qp_inner_failures`, `n_ls_failures`, `n_lbfgs_skips`, `n_nan_directions`, `max_gamma`, `min_diag`, `max_diag`, `eq_jac_min_sv_est`, `ls_alpha_min`, `tail_ls_failures`, `n_bound_fix_solves`, `max_bound_fixed`, `max_active_ineq`, `n_merit_regressions`, `n_proj_refinements` (cumulative MINRES-QLP M-metric refinement rounds), `max_proj_residual` (high-water `||A d - b||` after refinement), `n_divergence_blowups` (cumulative blowup-counter increments), and `divergence_triggered` (latched `True` once best-iterate rollback fires). The public helper `slsqp_jax.get_diagnostics(state)` returns the accumulator. Because the `qp_optimal & primal_feasible` shortcut has been removed from `terminate()`, these counters are the recommended way to distinguish a genuine `RESULTS.successful` from a run that bailed out early or fought L-BFGS the whole way through: a large `n_qp_inner_failures` combined with a small `eq_jac_min_sv_est` signals a near-rank-deficient equality Jacobian; a non-zero `tail_ls_failures` at termination means the returned iterate is a best-merit point, not a stationary point; `n_merit_regressions > 0` warns that the line search accepted a step whose merit value increased (either `rho` is undersized or the Hessian approximation is very stale).

The `n_lbfgs_skips` counter is computed from `slsqp_jax.lbfgs_should_skip(s, y)` (the same predicate used inside `lbfgs_append`) rather than diffing the post- and pre-append `count` field. The older diff-based check misfired once the ring buffer saturated at `memory`, reporting a skipped pair on every subsequent iteration even when the append actually wrote to the circular buffer. The helper is exported so downstream code can reproduce the decision without touching private fields.

The built-in `verbose=True` callback additionally prints `|∇L|/|L|` (the exact relative-stationarity criterion used by `terminate()`), `Δmerit` (signed change vs best-so-far), `skip` (was the curvature pair skipped), `#fix` (box-bound variables pinned this step), `fix#` (non-trivial bound-fixing inner solves used this step), and `LS tail` (consecutive line-search failure counter). These make it possible to distinguish "active-set churn" (oscillating `#fix`), "bad inner direction" (persistent `skip`/`LS ok: False`), and "we are simply running out of iterations" (`|∇L|/|L|` decreasing monotonically but still above `rtol`) without instrumenting the step function.

**Bound-fix inner-solve short-circuit:** The bound-handling loop in `_solve_qp_subproblem` performs up to 5 reduced-space inner solves per outer step. Previously each of those five passes called `inner_solver.solve(...)` unconditionally, and the result was only kept when `any_change & any_fixed` was true — wasteful on bound-heavy problems like `Portfolio(n=5000)` with `MinresQLPSolver`, where we were paying for `5 * max_steps` full `(n+m)`-dimensional MINRES-QLP solves whose results were discarded. The loop now wraps the inner solve in `jax.lax.cond(any_change & any_fixed, do_solve, skip_solve)` (the `skip_solve` branch returns the current `direction`/`multipliers` unchanged), and records the number of non-trivial solves in `QPResult.bound_fix_solves` (summed into `diag.n_bound_fix_solves`). The accepted direction is unchanged; the total number of MINRES-QLP/CG invocations drops dramatically (on the portfolio benchmark from `2500` to `339`).

## The Task: SLSQP Implementation
You need to port the logic of SLSQP (Sequential Least Squares Programming) to JAX.
* **Core logic:** SLSQP solves a nonlinear optimization problem by solving a Quadratic Programming (QP) subproblem at each step to determine the search direction.
* **Subproblem:** Since JAX/Optimistix does not have a native QP solver for this specific task, you may need to implement an **Active Set method** or a **Projected Gradient method** to solve the QP subproblem at each iteration.
* **Merit Function:** Use an L1-merit function (Han-Powell) to accept/reject steps, similar to the reference implementation.
